广发银行面试

发表于 2019-03-15 | 更新于: 2019-03-15 |

严格来说，这次是第一次面试，面试过程是先笔试后面试，笔试时间为60分钟，然后就是等待面试。和面试官谈了30分钟左右吧，谈到最后就感觉凉了。所以反思一下，以免同样的错误下次再犯。

笔试问题

笔试总分为100分，60分的选择题，40分的填空题，没有编程题。选择题考察范围很广，本科阶段计算机科学与技术专业和软件工程专业的必修课程都有考到，还涉及到测试的几个问题。填空题主要是代码填空，全部是C++代码。第一题是查找1000以内的回文素数，第二题是打印出指定行数的杨辉三角形，第三题是8*8的棋盘，马走国际象棋，通过深度搜索算法判断马是否能走完全部空格。

面试问题

面试问题主要是根据简历来问的，当时面试官主要问了我以下问题：

Java

JDBC连接步骤
JDBC返回结果集类型
文本文件读取
多线程实现方式有哪几种
线程同步有几种方式
java如何实现银行家算法
spring中bean配置文件单例和多例，多例模式用处
Tomcat使用
Linux常用命令有哪些

Android

Android activity生命周期

算法

二叉树遍历
哈希算法

Delphi

Delphi和Java的区别
Delphi如何创建对象和创建类
Delphi和Java是否可以实现多继承
一个Unit里是否可以写多个类
Delphi串口通信如何实现

其它

是否考研
为什么想来广发银行

面试总结

在面试之前，并没有去准备面试相关的东西，面试过程中，太紧张了，好几个问题都没有回到位。然后这个面试官给我的感觉是从Delphi转到Java开发的，一上来就问了好几个Delphi相关的问题，在茫茫人海中碰到一个用Delphi的人，还是倍感亲切的。但是问的和Delphi相关的问题并没有回答的很好，半年没有用到Delphi了，如果不看一下以前的代码，重新写程序还真的写不出来了。最气的就是问我Delphi的对象是怎么实例化的，我回答的是new出来的，现在想想，面试官估计怀疑我是不是写过Delphi吧，这么基础的问题都没回答出来，当时想到了Create()构造函数，但是中了Java的毒，崩溃。Java开发好几年没用过JDBC了，所以问到这个问题的时候，就有点懵了，勉强在面试官的提示下想起来了，接下来又问了我JDBC查询返回的结果集是什么类型的，然后就懵了，现在开发都是用的Mybaits,当时用JDBC的时候，也是调用的工具包，根本没有去关心里面具体是怎么实现的。然后还有文件读写的问题，这个除了当时项目开发的过程中用到过，平时都没有使用，所以也就没太认真去记到底是怎么实现的。这两个问题没回答好，这次的面试也就感觉凉了。然后还问了我碰见问题一般是怎么处理的，我当时设想的场景是第一次接触某个东西，然后不会使用，是怎么处理的。我想面试官应该是问我Debug能力吧。整场面试下来就懵了，面试官说不要灰心，好好学好基础。

并不灰心，只是有些可惜，这些基础的东西都没有回答到，这份工作还是比较向往的。然后就是感觉到了基础的重要性，之前也做过几次面试题，主要考察的都是基础，而我的学习风格是我想要完成什么任务，然后就会去网上查找相关技术再学习，从来没有哪门语言是系统性的学完的。这里简单回忆一下笔试和面试的一些题目。
